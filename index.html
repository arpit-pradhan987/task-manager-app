📚 Understanding Event Propagation

🫧 Event Bubbling

Events start at the target element and bubble up through parent elements.

Order: Child → Parent → Grandparent

This is the default behavior. Used with addEventListener()

⬆️ Event Capturing

Events start at the root and travel down to the target element.

Order: Grandparent → Parent → Child

Enabled by passing true or {capture: true} as third argument

📄 HTML Parsing

The browser receives HTML as a stream of bytes and converts it into a tree of DOM nodes. This happens character by character.

🔍 Tokenization

The parser breaks the HTML into tokens (tags, attributes, content). Each token represents a meaningful unit of the document.

🌳 DOM Tree

Tokens are converted into a tree structure representing the document. Every element, text node, and comment becomes a node in the tree.

🎨 CSSOM Tree

CSS is parsed similarly to HTML, creating the CSS Object Model. This represents all styles applicable to each element.

✨ Render Tree

Combines DOM and CSSOM. Only includes visible elements (hidden elements, display:none, etc. are excluded).

📐 Layout (Reflow)

Browser calculates exact positions and sizes of all elements. Changes to dimensions or positions trigger a new layout pass.

🖌️ Paint (Rasterization)

Browser converts the render tree into pixels. Different elements may be painted to separate layers for optimization.

🎬 Compositing

All layers are combined and displayed to the screen. GPU acceleration can be used for composite operations.
